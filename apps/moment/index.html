アプリ名

MOMENT

コンセプト

スクリーンタイムのプロフェッショナルな管理をシンプルなUI・日本語で。現状存在するOpalの機能を日本向けにリリースする。

ターゲット/目的

「第一志望に合格したいと思っている、高校3年生」の「スマホをいじって時間を潰してしまった自己嫌悪や焦りという苦痛を解決するアプリ」

コア価値

バグが少なく、エネルギー消費もない。一番シンプルで高機能。

収益化

モデル: フリーミアム

無料版

  • シンプルなタイマー設定機能
  • 選択したアプリをブロックする機能
  • 集中できた時間の合計を記録・表示
  • スケジュール機能（個数無制限）
  • ホワイトリスト機能（個数無制限）
  • よりシンプルな実績記録

有料版（プロ）

ディープフォーカスモード:

  • タイマー画面から簡単にオン/オフ
  • セッション中の通知を完全にシャットアウト
  • ホワイトリスト機能（個数無制限）
  • アプリのアンインストールによる解除の防止
  • 緊急停止機能（検討中）

高度な実績記録:

  • 過去の記録を全て遡って確認
  • 集中時間の詳細な分析（グラフ表示など）

機能優先度

P0：必須 (MVP)

  • タイマー設定機能（分単位）
  • ブロック対象アプリの選択機能
  • フォーカス・セッションの開始/終了機能
  • セッション中の強制ブロック機能（中断不可）
  • 集中できた時間の合計を記録・表示

P1：高

  • スケジュール機能（自動開始）
  • ホワイトリスト機能
  • ウィジェット機能
  • 詳細な実績レポート（日/週/月）

P2：中

  • ポモドーロタイマー
  • 環境音（雨音、焚き火など）
  • デザインテーマ（ライト/ダーク他）

P3：低（当面やらない）

  • フレンド機能、SNS連携
  • ゲーム要素（キャラクター、ランキング）
  • Android版

MVPまでの手順

  1. プロジェクト設定とUI骨格（1-2週間）: SwiftUIで主要4画面のガワを作成
  2. Screen Time API統合（2-3週間）: 単体アプリのブロックから開始
  3. セッションロジック実装（1-2週間）: タイマー×複数アプリブロック、中断不可
  4. データ永続化と実績表示（1週間）: SwiftDataで保存し表示

U/UXと画面構成

  1. ホーム: 大きな「集中を開始」ボタン、設定導線、今日の合計
  2. セッション設定: 時間ピッカー、アプリ選択、保存
  3. セッション中: 残り時間のみを大きく表示、黒基調
  4. 実績: 日/週/月の合計とカレンダー可視化

ユーザーフロー: 起動 → 開始 → 設定 → 開始 → セッション → 通知 → 実績

技術選定

  • Swift（パフォーマンスと安定性、OS連携）
  • SwiftUI（宣言的UIで速度向上）
  • Screen Time API（OSレベルブロックで確実・省電力）
  • SwiftData（少ないコードで安全な永続化）

開発の心得

  1. 完璧より完了（Done is better than perfect）
  2. 作っては触るのサイクルを最短に
  3. 次にやることをひとつだけに絞る
